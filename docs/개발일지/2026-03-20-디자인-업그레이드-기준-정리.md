# 디자인 업그레이드 기준 정리

- 작성시각: 2026-03-20
- 해결하고자 한 문제: Stitch MCP와 `design/` 스크린샷 기준으로 현재 앱을 어떤 화면 구조와 디자인 언어로 업그레이드해야 하는지 기준이 흩어져 있었음.

## 진행 내용

- `memory.md`를 다시 확인해서 이전 세션 맥락을 정리함.
- Stitch MCP로 프로젝트와 주요 스크린 메타데이터를 조회함.
- 사용자가 추가한 `design/dashboard.png`, `design/analysis.png`, `design/diary.png`, `design/journal.png`, `design/prediction.png`를 직접 확인함.
- 현재 `/emotion` 라우트가 어떤 UI였는지 코드 기준으로 다시 확인함.
- 위 내용을 바탕으로 `design/design.md`에 디자인 업그레이드 기준서를 작성함.

## 해결된 것

- 어떤 이미지가 어떤 화면에 대응되는지 정리됨.
- 기존 `/emotion` 디자인이 신규 디자인 기준에서 유지 대상이 아니라는 점이 정리됨.
- 모바일 퍼스트, 파스텔 카드, 네이비 헤더, 일기/저널 플로우까지 포함한 전체 디자인 방향이 문서화됨.

## 아직 안 된 것

- 실제 코드 반영은 아직 시작하지 않음.
- `prediction.png`로 파일명을 정리한 뒤 문서 반영까지 완료함.
- 신규 탭 구조와 실제 라우트 연결 구현은 아직 안 함.
