# 공개 API 보안 하드닝 및 debug 차단

- 작성시각: 2026-03-21 03:50:00 +09:00
- 해결하고자 한 문제: 공개 market API에 rate limit이 없었고, debug endpoint가 쿼리스트링으로 열릴 수 있었으며, 서버 내부 오류 메시지가 그대로 클라이언트에 노출되고 있었음. KRX 기본 endpoint도 HTTP로 잡혀 있어 키 전송 보안이 약했음.

## 진행 내용

- `src/server/request-security.ts`를 추가해서 IP 기반 rate limit 공용 로직을 만들었음.
- `src/server/opendart.ts`, `api/opendart/market-summary.ts`, `api/opendart/stock.ts`에 rate limit을 적용했음.
- `src/server/opendart.ts`에서 `/debug` endpoint를 development 환경에서만 열리도록 제한했음.
- `src/components/AdminDebugPanel.tsx`를 DEV 전용으로 제한하고 에러 표시를 일반화했음.
- 서버/API 응답에서 내부 `error.message` 대신 일반화된 에러 문구를 사용하도록 바꿨음.
- `src/server/krx.ts`의 KRX endpoint 기본값을 HTTPS로 바꾸고 비-HTTPS URL을 거부하도록 했음.
- `src/server/opendart.test.ts`에 debug 차단과 rate limit 테스트를 추가했음.

## 해결된 것

- 공개 API 남용에 대한 1차 방어선이 생겼음.
- debug 정보가 프로덕션에서 우회 노출되는 경로를 막았음.
- 내부 연동 오류가 브라우저에 그대로 노출되지 않게 됐음.
- KRX API 키가 기본 설정에서 평문 HTTP로 전송되는 문제를 막았음.
- `pnpm test -- --run`, `pnpm typecheck`, `pnpm lint`를 모두 통과했음.

## 아직 안 된 것

- 현재 rate limit은 메모리 기반이라 인스턴스가 여러 개인 배포 환경에서는 전역 보장이 없음.
- admin/debug를 장기적으로 다시 열려면 별도 인증 체계가 필요함.
