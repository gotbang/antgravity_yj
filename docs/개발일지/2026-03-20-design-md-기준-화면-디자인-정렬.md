# design.md 기준 화면 디자인 정렬

- 작성시각: 2026-03-20 23:49:07 +09:00
- 해결하고자 한 문제: 내용 구조는 정리됐지만 실제 화면이 `design/` 스크린샷과 `design/design.md` 기준의 모바일 카드 톤, 간격, 탭, 헤더 형태와 아직 완전히 맞지 않았음.

## 진행 내용

- `design/dashboard.png`, `design/analysis.png`, `design/prediction.png`, `design/diary.png`, `design/journal.png`를 기준으로 모바일 레이아웃과 카드 위계를 다시 확인함.
- `design/design.md`의 라운드, 색상 팔레트, 상단 네이비 헤더, 파스텔 카드, 모바일 고정폭 기준을 CSS에 반영함.
- 홈, 종목 분석, 예측 상세, 일기, 저널 루트에 화면별 클래스명을 부여해서 스크린샷 기준 조정 폭을 분리함.
- 모바일에서 기존 그리드가 1열로 무너지는 규칙을 줄이고, 홈 3개 진입 카드, 시장 감정 2열 카드, 예측 상세 2열 요약 카드가 유지되도록 수정함.
- 검색창, 탭, 종목 리스트 카드, 날짜 칩, FAB, 저널 하단 탭 바 등 반복 UI의 모양과 그림자 톤을 재보정함.

## 해결된 것

- `design.md` 기준의 모바일 퍼스트 카드 톤이 더 강하게 반영됨.
- 홈/분석/예측 화면의 카드 위계와 2열 구조가 스크린샷에 더 가까워짐.
- 일기/저널 화면의 칩, 플로팅 버튼, 하단 탭 바가 스크린샷 기준에 더 가까워짐.
- `pnpm test -- --run`, `pnpm typecheck`, `pnpm lint`를 모두 통과함.

## 아직 안 된 것

- 로컬 브라우저 자동 캡처가 막혀 있어서 최종 픽셀 단위 비교는 아직 못 했음.
- `/emotion` 라우트를 design.md 기준으로 완전히 재편할지, 현재 구조를 유지할지는 별도 결정이 필요함.
