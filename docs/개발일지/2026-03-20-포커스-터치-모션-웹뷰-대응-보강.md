# 포커스, 터치, 모션, 웹뷰 대응 보강

- 작성시각: 2026-03-20
- 해결하고자 한 문제: 실제 포커스 스타일, 터치 타깃 크기, reduced motion 대응, 모바일 앱 웹뷰 안전 영역 처리가 부족했음.

## 진행 내용

- 셸 내부 `Suspense` fallback을 레이아웃 유지형 스켈레톤으로 교체함.
- `StockPage` 검색 입력에 명시적 라벨을 추가하고 토글 버튼 그룹 semantics를 정리함.
- 주요 인터랙티브 요소에 `:focus-visible` 스타일을 추가함.
- 별표 버튼, 하단 탭, 월 이동 버튼, 플로팅 버튼, 날짜 칩 등 터치 타깃을 보강함.
- `prefers-reduced-motion` 대응과 스켈레톤 애니메이션 제어를 추가함.
- `100dvh`, safe area inset, overscroll-behavior, tap highlight 제거 등 웹뷰 대응 스타일을 추가함.
- 사용자 노출 텍스트를 다시 정리해서 접근성 이름과 시각 텍스트를 안정화함.
- `pnpm test`, `pnpm typecheck`, `pnpm lint`를 실행함.

## 해결된 것

- 키보드 포커스가 실제로 보이게 됨.
- 모바일 앱 웹뷰에서도 상하 safe area를 더 안전하게 처리함.
- 저모션 환경에서 과한 애니메이션이 줄어듦.
- 검색/토글 UI 접근성이 개선됨.

## 아직 안 된 것

- 실제 디바이스 웹뷰 스크린샷 기준의 미세 보정은 추가로 진행할 수 있음.
