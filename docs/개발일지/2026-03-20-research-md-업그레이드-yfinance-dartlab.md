# research.md 업그레이드: yfinance 제약 + dartlab 포함

- 작성시각: 2026-03-20
- 해결하고자 한 문제: 무료 데이터 소스를 공개 서비스에 쓸 수 있는지 명확하지 않았고, `dartlab`을 어떤 레이어 후보로 봐야 하는지 정리할 필요가 있었음.

## 진행 내용

- 리포 안에 기존 API 키나 실데이터 소스 흔적이 있는지 먼저 확인함.
- `yfinance`, `Twelve Data`, `KRX OPEN API`, `한국투자증권 Open API`, `네이버 검색 API`, `OpenDART`, `dartlab` 관련 최신 공식 문서와 안내 페이지를 다시 확인함.
- 무료 여부와 공개 서비스 허용 여부를 분리해서 판단함.
- `research.md`를 새로 작성해 시세/지수, 뉴스, 공시/재무, 감정/해석 레이어로 비교 구조를 재정리함.
- `dartlab`을 시세 API 후보가 아니라 공시 분석 파이프라인 후보로 별도 섹션에 정리함.

## 해결된 것

- `yfinance`는 무료라도 공개 서비스 주력 소스로 쓰면 안 된다는 점을 명시함.
- `dartlab`은 후속 공시 해석/요약 워커 후보라는 점을 명시함.
- 현재 제약에서 바로 구현 가능한 범위와 보류할 범위를 문서로 구분함.

## 아직 안 된 것

- `research.md` 승인 전이라 실제 실데이터 연동은 시작하지 않음.
- 무료 공개 서비스용 시세 주력 소스는 아직 확정하지 않음.
