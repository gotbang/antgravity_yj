# KRX 시세 연동 준비 및 공통 브랜드 헤더 적용

- 작성시각: 2026-03-21 01:00:00 +09:00
- 해결하고자 한 문제: KRX OPEN API 키가 발급되면 추가 코드 수정 없이 가격 시세가 바로 화면에 노출되도록 준비하고, 업로드한 레퍼런스 이미지 톤에 맞춘 브랜드 헤더를 모든 페이지에 동일하게 적용해야 했음.

## 진행 내용

- `StockPrediction` 타입에 가격 표시용 필드를 추가해서 공시/재무 데이터와 KRX 시세를 함께 담을 수 있게 정리함.
- `src/server/krx.ts`를 새로 만들어 KRX OPEN API 일별 시세 응답을 읽고 종목 코드 기준으로 매핑하는 로직을 추가함.
- `src/server/opendart.ts`에서 OpenDART 결과에 KRX 가격 정보를 병합하도록 수정하고, KRX 실패 시에도 기존 공시/재무 데이터만으로 안전하게 동작하도록 fallback을 넣음.
- `src/server/opendart.test.ts`를 추가해서 `KRX 키가 있을 때 시세 병합`, `KRX 실패 시 기존 데이터 유지` 시나리오를 테스트함.
- 홈, 종목, 감정, 일기, 저널 화면 상단을 공통 `BrandHeader`로 교체해서 업로드한 레퍼런스와 비슷한 네이비 헤더 + 둥근 배지 톤으로 통일함.
- 공통 헤더에 네비게이션을 넣고 브랜드 영역과 `홈` 탭을 통해 첫 화면(`/`)으로 이동할 수 있게 수정함.
- 종목 리스트/상세/홈 관심종목 영역에 가격, 등락, 등락률 표시를 추가함.

## 해결된 것

- KRX 키만 주입되면 종목 화면과 홈 관심종목에서 가격/등락 정보가 바로 노출되는 코드 경로를 준비함.
- KRX 연동이 실패하거나 키가 없어도 기존 OpenDART 기반 화면은 깨지지 않게 유지됨.
- 모든 페이지 상단 헤더가 같은 브랜드 스타일과 네비게이션 구조로 통일됨.
- `pnpm test`, `pnpm typecheck`, `pnpm lint`, `pnpm build`를 모두 통과함.

## 아직 안 된 것

- 실제 KRX OPEN API 인증키가 아직 없어서 외부 실서버 호출 검증은 이번 단계에서 못 함.
- KRX 승인 결과에 따라 서비스 URL이 다르면 `KRX_KOSPI_DAILY_API_URL` 환경변수로 맞춰서 한 번 더 실검증해야 함.
- 업로드 이미지 원본 자체를 앱 에셋으로 직접 쓰는 작업은 별도 원본 파일 확보가 필요함.
