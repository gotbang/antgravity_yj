# tokens.css app.css 분리 및 톤 마감

- 작성시각: 2026-03-21 00:35:10 +09:00
- 해결하고자 한 문제: `src/index.css` 하나에 토큰과 실제 컴포넌트 스타일이 함께 섞여 있어서 유지보수가 어려웠고, 전체 페이지 톤도 분리된 구조 안에서 마지막으로 통일할 필요가 있었음.

## 진행 내용

- `src/index.css`를 진입점 역할만 하도록 축소하고 `src/styles/tokens.css`, `src/styles/app.css`로 분리함.
- `tokens.css`에는 컬러, 반투명 표면, 텍스트, 섀도우, 라운드, 간격 토큰과 `.dark` 변수를 정리함.
- `app.css`에는 실제 레이아웃, 카드, 배지, 탭, 리스트, 차트, FAB, 저널 하단 바 등 구현 스타일을 이동함.
- 리터럴 색상 제거 작업과 함께 전체 페이지의 공통 톤을 유지하도록 재정리함.
- 분리 이후 `pnpm test -- --run`, `pnpm typecheck`, `pnpm lint`를 재실행해 이상이 없는지 확인함.

## 해결된 것

- 스타일 유지보수 구조를 `tokens.css + app.css + index.css(import 전용)`로 정리함.
- 홈/stock/detail/diary/journal 전체 톤이 분리된 구조 안에서도 유지됨.
- 검증 명령어 모두 통과함.

## 아직 안 된 것

- 페이지 단위 스타일 파일 추가 분리
- 프리뷰 페이지가 실제 앱 토큰 파일을 직접 참조하도록 연결
