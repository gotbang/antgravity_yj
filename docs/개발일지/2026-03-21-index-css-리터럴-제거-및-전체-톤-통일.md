# index.css 리터럴 제거 및 전체 톤 통일

- 작성시각: 2026-03-21 00:31:24 +09:00
- 해결하고자 한 문제: `src/index.css`에 남아 있던 하드코딩 색상과 그림자 값을 걷어내고, 홈/stock/detail/diary/journal 전체 화면이 동일한 토큰 체계와 톤으로 보이게 정리해야 했음.

## 진행 내용

- `src/index.css`의 토큰 사전을 더 확장해서 표면색, 보조색, 상태색, 텍스트색, 반투명 흰색, 차트용 블루, 섀도우까지 세분화함.
- 헤더, 카드, 배지, 검색창, 탭, 전략 카드, 상태 pill, FAB, 저널 카드, 차트 배경 등 공통 UI의 리터럴 색/그림자를 토큰 참조로 치환함.
- `index.css` 본문 기준으로는 리터럴 색상이 사실상 사라지고, 토큰 정의 영역에만 값이 남도록 정리함.
- 전체 페이지가 더 일관된 파스텔 카드 톤과 네이비 헤더 톤을 유지하도록 통일함.

## 해결된 것

- `src/index.css` 본문에서 남아 있던 리터럴 색상 대부분 제거
- 홈/stock/detail/diary/journal 공통 톤 통일
- `pnpm test -- --run`, `pnpm typecheck`, `pnpm lint` 모두 통과

## 아직 안 된 것

- 각 페이지 전용 CSS를 별도 파일로 나누는 구조적 분리
- 토큰을 `src/styles/tokens.css` 같은 전용 파일로 추출하는 리팩토링
