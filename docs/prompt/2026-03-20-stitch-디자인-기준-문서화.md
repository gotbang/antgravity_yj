# Stitch 디자인 기준 문서화

- 작성시각: 2026-03-20
- 해결하고자 한 문제: Stitch 기준 화면과 사용자가 추가한 디자인 스크린샷을 바탕으로, 앞으로 구현할 UI의 단일 기준 문서가 필요했음.

## 사용한 프롬프트 요약

- `memory.md를 완벽히 이해하고 다 이해했으면 ok. 추가 질문사항 있으면 질문해`
- `그럼 이어서 스티치 mcp로 링크주소 읽어서 디자인 완벽히 이해. 그리고 질문`
- `design폴더에 바꿀려는 이미지 스크린샷 넣어놨어`
- `또 질문할거 있으면 질문하고 없으면 디자인 업그레이드 할 내용을 디자인 폴더 안에 design.md로 만들어줘`

## 해결된 것

- 디자인 판단 근거를 Stitch 프로젝트와 `design/` 폴더 스크린샷으로 고정했음.
- `design/design.md` 문서를 생성할 수 있는 수준으로 화면 구조를 정리했음.

## 아직 안 된 것

- 실제 화면 구현과 라우트 개편은 아직 진행하지 않음.
