# Ant Gravity MASTER

## 입력 이해

- 제품 타입: 모바일 우선 투자 도우미 웹앱
- 플랫폼: web
- 브랜드 캐릭터: 귀엽고 똑똑한 개미 투자 친구
- 레퍼런스 타입: 업로드 이미지 1장, 기존 `design/design.md`
- 산출물 목적: 앞으로 모든 화면이 공유할 단일 디자인 방향 정의

## 가정과 누락 정보

- 업로드 이미지는 홈 대시보드 한 장이지만, 색감과 컴포넌트 성격이 다른 화면까지 확장 가능한 수준으로 명확하다고 판단함.
- 실제 캐릭터 원본 에셋은 없는 상태이므로, 동일 무드의 둥근 개미 마스코트 규칙으로 일반화함.
- 다크 모드는 요구되지 않았지만 토큰 문서에서는 대응 가능한 수준으로 정의함.

## 브랜드 한 줄 요약

- `따뜻하고 둥근 감성으로 어려운 투자 정보를 쉽게 풀어주는 개미 친구 앱`

## 스타일 키워드

- `cute fintech`
- `storybook pastel`
- `soft card dashboard`
- `rounded mascot ui`
- `mobile-first comfort`

## 컬러 무드 요약

- 메인은 `짙은 네이비`로 신뢰를 만들고
- 본문 카드는 `하늘색, 버터 옐로, 베이비 핑크, 민트`로 감정과 기능을 분리한다
- 흰 배경 위에 파스텔 카드가 떠 있는 형태로 부담을 줄인다

## 폰트 무드 요약

- 제목은 둥글고 굵은 산세리프
- 본문은 부드럽지만 선명한 고가독성 산세리프
- 전체적으로 `장난스럽다`보다 `친근하고 안정적이다` 쪽이 우선

## 안티패턴

- 차가운 금융 대시보드처럼 회색과 남색만 쓰는 구성
- 얇고 날카로운 타이포그래피
- 작은 텍스트에 정보 과밀 배치
- 테이블 중심의 딱딱한 시세 화면
- 강한 유리 효과, 네온, 과한 블러
- 이모지 남용으로 UI가 유치해지는 구성
- 너무 많은 보더와 구분선으로 카드가 조각나는 레이아웃

## 레이아웃 규칙

- 모바일 기준 폭은 `390px 전후`
- 화면은 `상단 브랜드 영역 -> 핵심 액션 -> 감정/데이터 카드 -> 보조 전략` 순서로 흐른다
- 카드 간 간격은 `12px~20px`
- 한 섹션 안 카드 수는 많아도 `2열 또는 세로 스택`까지만 허용
- 스크롤은 자연스럽게 이어지되, 섹션 제목은 항상 충분한 여백을 갖는다

## 컴포넌트 규칙

### 브랜드 헤더

- 짙은 네이비 라운드 직사각형
- 좌측은 `원형 개미 캐릭터 일러스트`
- 우측은 큰 영문 로고 + 중간 크기 한글 서브카피
- 큰 라운드와 부드러운 그림자 유지
- 모든 주요 페이지에서 같은 헤더를 반복 사용한다

### 진입 카드

- 3개 카드가 동일 크기 그리드
- 각 카드 배경은 `하늘색`, `버터 옐로`, `핑크`
- 내부 아이콘은 카드 중앙 상단의 `일러스트형 SVG 아이콘`
- `오늘 판단`은 상승 화살표 카드, `종목 분석`은 막대 그래프 카드, `개미의 일기`는 하트가 있는 다이어리 카드로 고정한다
- 텍스트는 카드 하단 중앙, 한 줄 또는 두 줄 이내

### 정보 카드

- 기본 표면은 흰색 또는 매우 옅은 파스텔
- 강조 카드만 풀컬러 배경 사용
- 한 카드 안에는 핵심 메시지 1개만 크게 강조
- 숫자 카드에는 pill 또는 내부 캡슐 영역을 둔다

### 배지와 칩

- 전부 pill 형태
- 상태값은 초록, 회색, 핑크 계열로 구분
- 정보 칩은 텍스트를 1줄로 유지

### 버튼

- 작은 버튼도 최소 높이 `44px`
- 모서리는 충분히 둥글게
- CTA는 카드와 분리되는 독립 배경색 사용

## 그림자 규칙

- 그림자는 `부드럽고 아래로 짧게`
- 검정 그림자보다 `보라/회색 섀도우` 느낌
- 강한 elevation 한 단계만 사용
- 같은 화면 내 섀도우 세기는 거의 통일

## 타이포그래피 규칙

### 제목

- 두께 매우 굵게
- 자간은 약간 좁게
- 화면 제목은 시선을 강하게 끌되 2줄 넘기지 않기

### 본문

- 최소 `16px` 권장
- 줄간은 `1.45~1.65`
- 부가 설명은 `Muted Text` 계열이지만 대비는 충분히 유지

### 숫자

- 숫자는 카드의 핵심이면 크게
- 단위는 숫자보다 한 단계 작게
- 퍼센트, 점수, 날짜는 정렬이 안정적으로 보이게 배치

## 캐릭터 규칙

- 캐릭터는 둥근 얼굴과 큰 눈을 가진 `친근한 개미` 계열
- 실사 느낌 금지
- 금융 공포를 줄이는 정서적 완충 역할
- 헤더에서는 `원형 프로필 일러스트`처럼 사용
- 홈 진입 카드와 감정 카드에서는 캐릭터 대신 친근한 그림책형 아이콘과 표정 일러스트를 우선 사용
- 과한 반복 노출은 피하고, 브랜드 인지 포인트로만 사용

## 화면 적용 원칙

### 홈

- 레퍼런스 이미지의 감정 카드 조합을 가장 충실히 반영
- `오늘 판단`, `종목 분석`, `개미의 일기` 카드가 첫 액션이어야 함
- 상단 브랜드 헤더와 3개 진입 카드가 하나의 대표 인상으로 작동해야 함

### 종목 분석

- 내용은 실데이터 중심으로 가도 시각 톤은 홈과 동일
- 검색창, 탭, 리스트 카드 모두 더 둥글고 말랑해야 함
- 실시간 가격 미연동 시 이를 숨기지 말고 친절하게 설명
- KRX 시세가 붙으면 가격, 전일 대비, 등락률이 카드 안에 함께 보이는 구조를 허용함

### 종목 상세

- 흰 카드 중심으로 정보 집중
- 상단은 별도 탭 바보다 `공통 브랜드 헤더 + 상세 정보 카드` 흐름으로 자연스럽게 연결
- 확률바와 요약 카드가 먼저, 긴 설명은 그 다음

### 일기 / 저널

- 감정과 기록이라는 성격을 더 강조
- 금융보다 커뮤니티/다이어리 무드가 앞서야 함
- 그래도 상단 브랜드 헤더는 공통으로 유지해서 앱 전체 인상을 통일함

## 접근성 규칙

- 본문 텍스트 대비는 WCAG AA 기준 이상 유지
- 핑크/옐로 배경 위 검정 텍스트 우선
- 포커스 링은 파스텔 블루 계열로 명확히 표시
- 탭, 카드, FAB 모두 `44x44px` 이상 터치 타깃 유지

## 최종 방향

- 이 디자인 시스템의 핵심은 `복잡한 금융 정보를 귀여운 캐릭터와 파스텔 카드로 심리적으로 가볍게 만드는 것`이다.
- 모든 후속 화면은 새 스타일을 추가하기보다 이 문서의 톤과 컴포넌트 성격을 반복해 확장해야 한다.
